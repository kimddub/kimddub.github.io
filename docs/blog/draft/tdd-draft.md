Tdd
===========
# Concept
- Test Driven Development의 약자로 테스트가 개발을 주도한다라는 개념
- 방법
    - Given
        - 테스트를 위해 주어진 상태
        - 테스트 대상에게 주어진 조건
        - 테스트가 동작하기 위해 주어진 환경
    - When
        - 테스트 대상에게 가해진 어떠한 상태
        - 테스트 대상에게 주어진 어떠한 조건
        - 테스트 대상의 상태를 변경시키기 위한 환경
    - Then
        - 앞선 과정의 결과
- 선행
    - service > method 단일책임원칙(SRP)에 준수하여 구현
